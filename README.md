# نظام تنظيم الجوازات — v1.3

## نظرة عامة
نظام تشغيلي + إداري لتتبع جوازات الحجاج يعمل كسيرفر محلي (Flask) على شبكة داخلية بدون إنترنت.

## البنية الحالية (v1.0)
- `app.py` — سيرفر Flask كامل (واجهة + API)
- `data.json` — قاعدة البيانات الحالية
- يشتغل على Windows بأمر: `python app.py`
- الموظفين يفتحون الواجهة من المتصفح عبر الشبكة الداخلية

## الشركات الست
- فاغبين (fagh)
- امسا (amsa)
- كوبا (cuba)
- سفريات العمرة (safar)
- تروبك (top)
- توب ١٠ (top10)

## المستخدمين
- مشرف عام (admin) — صلاحيات كاملة
- موظف لكل شركة — صلاحيات محدودة بمجموعته

## المطلوب تطويره — v1.3

### 1. ترقية التخزين
- استبدال `data.json` بـ SQLite
- دعم التزامن (Concurrency)
- الحفاظ على البيانات الحالية عند الترقية

### 2. الأمان
- تشفير كلمات المرور (bcrypt hashing)
- نظام جلسات (Flask-Session)
- عدم تخزين كلمات المرور في الكود — تخزينها في DB
- تقييد الوصول للشبكة الداخلية

### 3. النسخ الاحتياطي
- نسخ تلقائي كل X ساعات في مجلد `backups/`
- الاحتفاظ بآخر 7 نسخ
- إمكانية الاسترجاع من الواجهة

### 4. نظام الصلاحيات
**المشرف:**
- صلاحيات كاملة على كل الشركات
- إضافة/تعديل/حذف أي سجل
- إدارة الموظفين (إضافة، تعديل، تعطيل)
- إدارة المجموعات (إضافة، تعديل)
- نقل الموظفين بين المجموعات
- تصدير البيانات
- عرض السجلات (Logs)
- إحصائيات أداء الموظفين

**الموظف:**
- إضافة/تعديل/حذف سجل واحد فقط في مجموعته
- بحث ومتابعة
- صفحة المغادرة

### 5. نظام المغادرة (منفصل)
- صفحة مستقلة للمغادرة
- بحث برقم الجواز → عرض البيانات → "تأكيد المغادرة"
- عند إضافة حاج → "لم يغادر" تلقائياً
- لا يمكن تغيير المغادرة إلا من صفحة المغادرة
- تسجيل وقت المغادرة

### 6. سياسة الحذف
- حذف سجل واحد فقط (مع تأكيد)
- ممنوع: حذف جماعي أو حذف مجموعة كاملة
- كل حذف يُسجَّل في Logs

### 7. السجلات (Audit Logs)
تسجيل كل عملية مع:
- اسم المستخدم + المجموعة
- نوع العملية (دخول/خروج/إضافة/تعديل/حذف/مغادرة)
- رقم الجواز
- القيم القديمة والجديدة
- الوقت

واجهة عرض للمشرف:
- ترتيب زمني (الأحدث أولاً)
- فلتر حسب: المستخدم / المجموعة / نوع العملية / رقم الجواز

### 8. إحصائيات أداء الموظفين
لكل موظف:
- عدد الإضافات / التعديلات / المغادرات / الحذف
- عرض حسب: يوم / أسبوع / شهر
- ترتيب حسب الأداء

### 9. تحسينات الداشبورد
- إحصائيات: إجمالي / مفعّل / غير مفعّل / غادر / متبقين
- أزرار سريعة: إضافة / بحث / تصدير / المغادرة
- نسبة المغادرة
- آخر العمليات

### 10. تحسين UX والتصميم
- تصميم زجاجي (Glass UI):
  - خلفيات شبه شفافة مع Blur
  - بطاقات زجاجية (Cards)
  - ظلال ناعمة
  - حواف دائرية
  - تدرجات لونية هادئة
- ألوان موحدة: أخضر=مفعّل / أحمر=غير مفعّل / برتقالي=غادر
- دعم الجوال
- Pagination للقوائم الكبيرة
- Sort للأعمدة
- وضع ليلي (اختياري)

## قيود مهمة
- يجب أن يعمل 100% بدون إنترنت
- ملف واحد قابل للتشغيل على Windows بأمر بسيط
- لا dependencies معقدة — يجب أن تُثبَّت بـ pip بسهولة
- الواجهة تشتغل من أي متصفح على الشبكة الداخلية
- حد أقصى 10,000 حاج و5 مستخدمين متزامنين
