# نظام الجوازات

نظام محلي (Local) لإدارة وتتبع جوازات الحجاج وإدارة رحلات المغادرة، مبني على **Flask + SQLite**، يعمل داخل الشبكة الداخلية بدون إنترنت.

---

## هيكل الملفات

```
app.py                         ← السيرفر الكامل (Python / Flask / API / Routes)
requirements.txt               ← مكتبات Python المطلوبة
README.md                      ← هذا الملف

install.bat                    ← تثبيت المتطلبات (أول مرة فقط)
run.bat                        ← تشغيل مباشر وبسيط

مراقب النظام.ps1              ← أيقونة system tray + health check

بيانات الدخول الأولية.txt     ← بيانات الدخول الأولية — للمسؤول فقط (ملف التسليم المعتمد)
                                  ملاحظة: كلمات السر.txt يُستبعد من نسخة التسليم النهائية

  flask-*.whl
```

## ملفات التشغيل

### install.bat
يُشغَّل **مرة واحدة فقط** عند أول تنصيب على جهاز جديد.
يثبّت المكتبات من `libs/` إذا كانت متوفرة (بدون إنترنت)، وإلا يثبّتها من الإنترنت تلقائياً.

### run.bat
يشغّل النظام بشكل مباشر وبسيط — يفتح المتصفح تلقائياً على `http://127.0.0.1:5000`.
مناسب للاستخدام اليومي.

### تشغيل النظام.bat
يشغّل النظام في الخلفية (hidden window)، ينتظر حتى يستجيب الخادم، يفتح المتصفح، ثم يشغّل المراقب.

### مراقب النظام.ps1 + monitor.bat
يضع أيقونة في system tray، يراقب الخادم كل 5 ثوانٍ، ويعرض إشعار إذا تعطّل.
يحتوي قائمة سياق: فتح المتصفح، إيقاف الخادم، إغلاق المراقب.

## التشغيل على Windows

### أول مرة (تنصيب المتطلبات)

**مع إنترنت:**
```
1. تثبيت Python 3.10 أو أحدث من python.org (فعّل "Add to PATH")
2. نقر مزدوج على: install.bat
3. نقر مزدوج على: run.bat
4. تسجيل دخول: admin / admin2024
```

**بدون إنترنت (من مجلد libs):**
```
1. تثبيت Python مطابق لإصدار ملفات libs (انظر اسم ملف markupsafe للإصدار)
2. نقر مزدوج على: install.bat
   (سيكتشف libs تلقائياً ويثبّت منها)
3. نقر مزدوج على: run.bat
```

> **تنبيه:** ملف `markupsafe-*.whl` في `libs/` مرتبط بإصدار Python وبنية الجهاز (مثال: `cp314-win_amd64`).
> إذا اختلف إصدار Python على الجهاز الجديد، فقد يلزم إعادة تحضير ملفات libs أو التثبيت من الإنترنت.

### تشغيل يومي
```
نقر مزدوج على: run.bat
أو: تشغيل النظام.bat (للتشغيل مع مراقبة تلقائية)
```

## الشاشات والمسارات

### /scan — الإدخال السريع
صفحة مستقلة لإدخال أرقام الجوازات بسرعة عبر ماسح ضوئي أو لوحة المفاتيح.
**الوظيفة الوحيدة:** تسجيل الجواز (add) — لا يُغيّر حالة المغادرة.

### /scan?mode=depart — المغادرة الفردية
نفس الصفحة لكن في وضع المغادرة.
**الوظيفة الوحيدة:** تسجيل مغادرة جواز واحد — يحوّل `departed` إلى `نعم`.

> **مهم:** لا يوجد في /scan أو /scan?mode=depart أي ربط بنظام الرحلات.
> المغادرة عبر /scan فردية مباشرة، ومستقلة عن الرحلات تماماً.

### s-trips — إدارة الرحلات (داخل الواجهة الرئيسية)
شاشة إدارة رحلات المغادرة الجماعية. متاحة للمشرف وللموظف (بصلاحيات محدودة).

## سير عمل الرحلات

```
1. إنشاء رحلة          ← admin أو worker (مجموعته فقط)
                          الحالة: "مسودة"
                          لا يُغيّر departed لأي حاج

2. ربط حجاج بالرحلة    ← إضافة / إزالة حجاج من المجموعة المرتبطة
                          لا يُغيّر departed لأي حاج

3. إصدار تقرير Word     ← تصدير تقرير رحلات اليوم أو تقرير الرحلات المغادرة
                          لا يُغيّر departed لأي حاج

4. تأكيد الرحلة         ← admin أو worker (صاحب الرحلة)
                          الحالة: "مؤكدة"
                          لا يُغيّر departed لأي حاج

5. تأكيد مغادرة الرحلة  ← admin أو worker (مجموعته فقط، يشترط أن تكون الرحلة مؤكدة)
                          الحالة: "غادرت"
                          يحوّل departed='نعم' لجميع الحجاج المرتبطين بالرحلة
```

> **الخلاصة:** الخطوات 1 إلى 4 لا تؤثر على حالة `departed` أبداً.
> فقط **تأكيد مغادرة الرحلة** (الخطوة 5) هو الذي يُعدّل حالة المغادرة للحجاج.

## ميزات خاصة

### نظام الملاحظات + المرفقات
الملاحظات نص عادي مع دعم مرفقات بصيغة:
```
[[file|اسم الملف|mime/type|base64data]]
```
تُعرض كصور أو روابط تحميل حسب نوع الملف. تُخزَّن في حقل `notes` وفي مجلد `uploads/`.

### الجواز الدبلوماسي
يُكتشف تلقائياً إذا بدأ رقم الجواز بـ `DA`. يمكن للمشرف تعديله يدوياً.

### الرقم التسلسلي
- مستقل لكل مجموعة
- لا يُعاد استخدام الأرقام بعد الحذف (`MAX(seq) + 1` دائماً)
- صيغة: `PREFIX-رقم` (مثال: A-1، B-23)

### النسخ الاحتياطي
- تلقائي كل 6 ساعات (background thread)
- يحتفظ بآخر 7 نسخ فقط
- ملفات `.db` في مجلد `backups/`
- نسخة تلقائية قبل أي إعادة تهيئة

### الثيم الليلي / النهاري
- `localStorage` يحفظ الاختيار تحت مفتاح `ui-theme`
- الشعار: `logo.svg` (ليلي) ↔ `logo-day.svg` (نهاري)

## نقاط مهمة للمطور

1. **لا توجد مكتبات frontend** — vanilla JS فقط، لا React، لا Vue.
2. **`S.gid`** — كائن الجلسة في JS يستخدم `gid` وليس `group_id`.
3. **`can_act`** — `/api/search` يُرجع هذا الحقل: `true` إذا كان الجواز من مجموعة الموظف.
4. **Soft delete** — الحذف يضع `deleted=1` فقط. المشرف فقط يرى المحذوفين.
5. **تقارير Word** — تُولَّد بـ `zipfile` + OOXML مباشرة، بدون مكتبة python-docx.
6. **PORT** — افتراضياً `5000`، قابل للتغيير في السطر الأخير من `app.py`.
7. **RESET_PASSWORD** — متغير في `app.py` يحتوي كلمة سر تهيئة الموسم الجديد.
8. **بيانات الدخول** — احفظ ملف `بيانات الدخول الأولية.txt` في مكان آمن، لا تشاركه مع الموظفين.
